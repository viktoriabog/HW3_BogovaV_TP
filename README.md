## Дополнительное задание: веб-сервер для отчёта

В этом задании требуется запустить веб-сервер внутри Docker-контейнера, который раздаёт сгенерированный файл `report.html` так, чтобы пользователь мог открыть его в браузере своего локального компьютера при условии, что весь проект работает внутри GitHub Codespaces.

### В чем сложность и прикол (на понимание)

Этот проект склонируется не в проводник на компьютер, а в Codespaces - удаленную виртуальную машину в облаке GitHub. Сам сервер (контейнер с веб сервером nginx) запускается внутри этой удаленной машины, а браузер в свое время работает на локальном компьютере пользователя. Гитхаб дает возможность пробросить выбранный порт (у меня 8080), так что Codespaces начинает слушать этот порт и создает публичный URL `https://<уник индентификатор>...app.github.dev/` - по этой ссылке можно перейти из любого браузера (и трафик от браузера попадает в запущенный контейнер пройдя при этом через инфраструктуру гитхаба).

Что происходит пошагово:

```
Браузер (локальный компьютер)
    ↓ (HTTPS)
Публичный URL от GitHub Codespaces (порт 8080 в моем случае)
    ↓ (внутренний прокси GitHub)
Codespaces VM (удалённый хост) → порт 8080 на хосте
    ↓ (проброс портов Docker)
Docker-контейнер (веб-сервер nginx) → порт 80 внутри контейнера
    ↓ (монтирование тома)
Файл /usr/share/nginx/html/report.html (лежит в папке data/ на хосте Codespaces)
```
 
### Реализация в этом проекте

В файле `run.sh` команда `report_server` выполняет:

```bash
    docker run --rm \
      -v "$PROJECT_DIR/data:/usr/share/nginx/html:ro" \
      -v "$PROJECT_DIR/reporter/nginx.conf:/etc/nginx/conf.d/default.conf:ro" \
      -p 8080:80 \
      nginx:alpine
    ;;

```
Объяснение:

Запускаем контейнер `nginx:alpine`:
- `-v data/:/usr/share/nginx/html:ro` – подключает папку `data` на хосте внутрь контейнера (только чтение, на всякий случай), где nginx ищет файлы для отдачи.
- `-v reporter/nginx.conf:/etc/nginx/conf.d/default.conf:ro` – подменяет конфиг nginx своим, чтобы сервер отдавал `report.html` вместо `index.html`.
- `-p 8080:80` – пробрасывает порт 80 контейнера на порт 8080 хоста.
- `--rm` – удаляет контейнер после остановки.
P.S. (обьяснение конфиг файла) Так как внутри контейнера nginx по умолчанию ищет `index.html`, а у меня файл называется `report.html`, директива `index report.html;` добавлена через монтирование конфигурационного файла (reporter/nginx.conf). 

### Инструкция как открыть отчёт в браузере

1. **Откройте репозиторий в GitHub Codespaces**
   - Перейдите на страницу репозитория: [https://github.com/viktoriabog/HW3_BogovaV_TP]
   - Нажмите зелёную кнопку `<> Code` → вкладка `Codespaces` → `Create codespace on main`

2. **Сгенерируйте данные и отчёт**
   - В терминале Codespaces выполните:
     ```bash
     ./run.sh build_generator && ./run.sh run_generator
     ./run.sh build_reporter && ./run.sh run_reporter
     ```
   - В папке `data/` должны появиться файлы `data.csv` и `report.html`.

3. **Запустите веб-сервер**
   ```bash
   ./run.sh report_server
   ```
   - Далее должны вывестись сообщения о запуске контейнера. 

4. **Найдите порт в Codespaces**
   - Откройте вкладку Ports.
   - Там должен быть порт `8080`. 

5. **Откройте URL в браузере вашего компьютера**
   - Наведите курсор на строку с портом `8080` – появится значок глобуса или кнопка `открыть в браузере`.
   - Нажмите на неё. Откроется новая вкладка с адресом вида:
     ```
     https://<уник индентификатор>...app.github.dev/
     ```
   - Вы увидите HTML-отчёт, который был сгенерирован аналитиком.

   После закрытия Codespaces сервер остановится (тк URL доступен только пока работает Codespaces), при повторном открытии нужно будет снова запустить  `./run.sh report_server`.
